# AI User Simulator — App Details

## Vision

AI User Simulator is a web interaction agent that behaves like a real first-time user and tests websites before actual users do.

The goal is not to generate fake AI scores or enterprise reports.

The goal is simple:

> Help builders understand where users get confused, frustrated, or lost.

This tool is designed for:
- indie hackers
- startup founders
- hackathon teams
- frontend developers
- MVP builders
- student builders
- SaaS creators

The product should feel like:
> “Watching a real user interact with your app for the first time.”

---

# Core Product Philosophy

## 1. Build Fast, Ship Fast

The product is intentionally built in iterations.

We do not over-engineer before validation.

The process is:
- build prototype
- release quickly
- observe user behavior
- identify what users care about
- improve only validated features

If users do not strongly care about a feature:
- remove it
- simplify it
- move on

---

## 2. Real Utility Over AI Hype

This is NOT:
- an SEO checker

- an enterprise analytics dashboard

This IS:
- a realistic user simulation tool

- a first-impression testing system

The product must always prioritize:
- actionable feedback
- realistic interaction
- simplicity
- clarity
- usefulness

## 3. Word-of-Mouth Driven Growth

User Context Input

Before starting the test, the user should provide a short description about the app.

This helps the AI simulate:

realistic expectations
intended usage
natural user behavior

Without context, the AI may misunderstand:

product purpose
onboarding flow
user goals

Real users typically know:

what the product is for
why they opened it
what they are trying to achieve

The AI should simulate this realistic behavior.

Example Input
GitHub X-Ray helps recruiters and developers review GitHub profiles from a recruiter perspective.

or

This is a SaaS tool for helping students track hackathon projects and deadlines.
Purpose Of This Input

The description helps the AI:

understand intended product usage
evaluate whether the UX matches expectations
simulate realistic user intent
follow relevant workflows naturally
Important Rule

The AI should still evaluate:

clarity
onboarding
usability

Even if context is provided.

Example:
If the app description is clear,
but the website itself does NOT explain the product clearly,
the report should still mention that issue.
